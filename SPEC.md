# Spec: Recall вЂ” Developer Terminal Memory

## 1. Objective

**What:** Local-first CLI that captures shell commands, stores them in SQLite, and helps developers search their command history by project, time, or keyword.

**Why:** Developers forget commands they ran weeks ago. Shell history is chronological, not searchable, and dies when sessions close. Recall fixes this.

**User:** Software developers, DevOps engineers, CLI power users (Unix/Linux background, 22-40yo)

**Success (Phase 1 MVP):**
- `recall search <term>` returns matching commands in <100ms
- `recall recent` shows last 20 commands
- Shell hook captures commands without breaking terminals
- `recall init` completes onboarding in <2 minutes
- All data stored locally (`~/.recall/recall.db`)

### CLI Commands (Phase 1)
```bash
recall --help                    # Show usage
recall --version                 # Show version
recall init [--auto]            # Onboarding wizard
recall search "<term>"          # Search commands
recall recent [--limit 20]      # Recent commands
recall project                  # Current repo context
recall hook capture <payload>   # Internal: shell hook handler
recall uninstall [--keep-data] # Remove hooks
recall doctor                   # Debug installation
```

### Shell Hook (internal)
```bash
# Generated by recall init, added to ~/.zshrc or ~/.bashrc
eval "$(recall hook zsh)"       # For zsh
eval "$(recall hook bash)"      # For bash
```

### CLI Output (from ui/)

### Shell Hook Snippet Format

```typescript
// zsh: precmd + preexec
const zshSnippet = `
# Recall hook
precmdRecall() {
  recall hook capture --exit-code $? --duration-ms $RECVALL_DURATION --command-id $RECVALL_CMD_ID
}
preexecRecall() {
  RECVALL_CMD_START=$(date +%s%3N)
  RECVALL_CMD_ID=$(recall hook capture --raw-command "$1" --cwd "$(pwd)" --shell zsh --start-time $RECVALL_CMD_START 2>/dev/null | grep -o '[0-9]*')
}
autoload -Uz add-zsh-hook
add-zsh-hook preexec preexecRecall
add-zsh-hook precmd precmdRecall
`

// bash: PROMPT_COMMAND
const bashSnippet = `
# Recall hook
PROMPT_COMMAND="recall_hook_prompt"
recall_hook_prompt() {
  local exit=$?
  recall hook capture --exit-code $exit
}
```

## 6. Testing Strategy

### Test Framework
- **Bun test** вЂ” native test runner, `bun test`
- **Location:** `tests/` directory
- **Pattern:** `*.test.ts`

### Coverage Targets
| Layer | Type | Scope |
|-------|------|-------|
| Normalizer | Unit (TDD) | All normalization edge cases |
| History parser | Unit (TDD) | zsh/bash format parsing |
| Git detector | Unit | Edge cases (not git, nested, bare) |
| DB commands | Integration | CRUD operations |
| CLI commands | Manual | Search, recent, init flows |

### What to Test (Deterministic)
```typescript
// tests/import/normalizer.test.ts
import { describe } from 'bun:test'
import { normalize } from '../src/import/normalizer'

describe('normalize', () => {
  it('trims whitespace', () => {
    equal(normalize('  ls  '), 'ls')
  })
  it('collapses multiple spaces', () => {
    equal(normalize('ls   -la'), 'ls -la')
  })
  it('expands tilde', () => {
    equal(normalize('ls ~/foo'), `ls ${process.env.HOME}/foo`)
  })
})
```

### What NOT to Test (Manual/Integration)
- Shell hook behavior (spawns new shell)
- CLI flow (run commands, verify output)
- Installation (interacts with filesystem)

## 7. Boundaries

### Always Do
- [ ] Validate ALL shell input with Zod before processing
- [ ] Use parameterized queries (SQL injection prevention)
- [ ] Run `bun test` before `git commit`
- [ ] Handle `NO_COLOR` env var for CI safety
- [ ] Keep `sync/adapter.ts` interface intact for later swap
- [ ] Log to stderr (pino), output to stdout

### Ask First
- [ ] Adding new dependencies (check Bun compatibility first)
- [ ] Changing database schema
- [ ] Adding new CLI commands (affects UX surface)
- [ ] Changing `src/sync/adapter.ts` interface
- [ ] Modifying normalization rules (breaking change for users)

### Never Do
- [ ] Commit secrets, API keys, tokens
- [ ] Write to `~/.zshrc` or `~/.bashrc` without `--auto` flag
- [ ] Send data to external services in Phase 1-3
- [ ] Create daemon/process that runs in background
- [ ] Use `Bun.Glob` (not needed, git detection via `git rev-parse`)
